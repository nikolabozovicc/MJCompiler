# MicroJava Compiler

This project implements a compiler for the MicroJava programming language.  
The compiler translates MicroJava source code into bytecode executable on the MicroJava Virtual Machine (MJVM).

The compilation pipeline follows standard compiler phases: lexical analysis, syntax analysis, semantic analysis, and code generation.

## Compilation Phases

### 1. Lexical Analysis (JFlex)
Lexical analysis is implemented using **JFlex**.  
The lexer recognizes tokens such as identifiers, keywords, constants, and operators.

Path to `.lex` specification:  [MJLexer](./spec/mjlexer.lex/)

### 2. Syntax Analysis (CUP)
Syntax analysis is implemented using **CUP (LALR parser generator)**.  
The parser builds an abstract syntax tree (AST) based on the MicroJava grammar.

Path to `.cup` specification:  [MJLexer](./spec/mjparser.cup/)

### 3. Semantic Analysis
Semantic analysis is performed by traversing the AST and validating program correctness using a symbol table.

Implementation:  [SemanticAnalyzer](./src/rs/ac/bg/etf/pp1/SemanticAnalyzer.java/)

### 4. Code Generation
Code generation transforms the validated AST into MicroJava bytecode executable on the MJVM.

Implementation:  [CodeGenerator](./src/rs/ac/bg/etf/pp1/CodeGenerator.java/)

## Supported Features

The compiler supports the following language features:

- Global and local variables
- Enumerations (enum)
- Global constants
- Control flow (`if-else`)
- Ternary operator
- Arrays
- Operator `new`
- Input/output operations (`read`, `print`)
- Primitive types: `int`, `char`, `bool`
- Arithmetic expressions (`+`, `-`, `*`, `/`, `%`)
- Increment and decrement operators (`++`, `--`)
- Function calls (`ord(ch)`, `chr(i)`, `len(arr)`)

## Compiler Usage

The compiler is driven by the [Compiler.java](./test/rs/ac/bg/etf/pp1/Compiler.java/) class.

- Input: [program.mj](test/program.mj/)
- Output: `program.obj` (MicroJava bytecode)

The `.obj` file can be executed on the MicroJava Virtual Machine.

## Additional Modifications (Syntax Sugar)

In addition to the base specification, several syntax sugar features were implemented:

### Array print
- `print(arr)` - prints array elements
- `print(arr, i)` - prints i-th array element

### Array read
- `read(arr)` - reads array elements from input.txt

### Array maximum
- `#arr` - finds the maximum element of the array

### Array elements multiplicator
- `arr ^ NUM` - multiplies every array element with const NUM

### Array operator @
- `arr1 @ arr2` - returns new array that has the length of the bigger of those two. 
- `newArray[i] = arr1[i] + arr2[i]` - if arr1 and arr2 have different lengths, fill gap with zeroes

### Bool aray to integer
- Array of type bool is interpreted as int value. Elements of bool array can be 0 or 1 (false or true) nd those values represent binary representation of some number that will be formed here.
- `true false false true = 1 0 0 1 = 9`

### Final static array
- `final type arr[]` - value can be assigned to every element of the array only once. Only for static arrays

### Rotation of the array
- `rotateRight(niz, n)` - rotate array to right for n times

### Operator <=>
- `factor1 <=> factor2` - result is 1 if factor1 > factor2, 0 if equal, -1 if factor1 < factor2

### Operator ??
- `expr -> expr1 ?? expr2` - result is expr1 if expr1 != 0 and expr2 if expr1 == 0

## Build and Execution

The project uses **Apache Ant** for build automation.

### Build Script

The [build.xml](./build.xml/) file defines the compilation and execution targets. Two most important targets for starting the Compiler and executing MicroJava bytecode are:

- `compileMj` — runs the compiler and generates `program.obj`
- `runObj` — executes the generated bytecode on MJVM

## Input / Output Behavior

- Program input is read from: [test/input.txt](./test/input.txt/)

- Standard output is printed to console

- Debug output (stack state and instruction trace) is written to: `test/output.txt`
- Due to Ant execution constraints, interactive console input is not used.

### Structure of the program
```java
program testMJ

// global variables and constants

{
	void main()
	// local variables
	{
		// code
	}
}
```

## Setup and Running Instructions (Recommended)

### 1. Clone the repository

```bash
git clone https://github.com/nikolabozovicc/MJCompiler
cd MJCompiler
```
### 2. Install Apache Ant

Download and install from:
https://ant.apache.org/bindownload.cgi

After downloading:

- Extract the archive (ZIP) to a desired location, for example: `C:\apache-ant`

Set environment variables:

- `ANT_HOME = C:\apache-ant`

- `JAVA_HOME = <path_to_your_JDK_1.8> (Java 1.8)`

Add to PATH:

- `%ANT_HOME%\bin`
- `%JAVA_HOME%\bin`

Verify installation:
`ant -version`

### 3. Prepare input

Place your MicroJava source code in:
`/test/program.mj`

### 4. Compile the program

Run `ant compileMj`

### 5. Run the generated bytecode

Run `ant runObj`

## _Notes_

- Java 1.8 is recommended

- Input must be provided via input.txt

- Debug output is automatically generated in output.txt
